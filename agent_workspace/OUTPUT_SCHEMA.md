# Output Schema

This file records expected artifact shapes for validation. Existing scripts may include extra columns or keys; agents should tolerate extra fields but require the minimum fields below when the artifact exists.

## `fig_table_inventory.csv`

Minimum expectations:

- CSV parses with pandas or a standard CSV reader.
- Row count is greater than zero for papers with figures/tables.
- One identifier column exists: `item_id`, `pdf_item_id`, or `item`.
- Useful optional columns: `caption`, `label`, `type`, `page`, `source_file`.

## `fig_table_lnpdb_classified.csv`

Minimum expectations:

- CSV parses.
- It preserves an FT identifier column.
- It contains some relevance/classification signal.
- After manual review, it should contain `manual_select`.

Useful optional columns:

- `is_lnpdb`
- `lnpdb_class`
- `reason`
- `manual_select`
- `excel_item_id`
- `matched_blocks`
- `matched_block_csv_path`

## `.manual_select_review_done`

Minimum expectations:

- File exists in the paper folder.
- Its content may be empty; existence is the gate for active agent stages.

## `total_figure_mapping.json`

Expected shape:

```json
{
  "source_folder_name": {
    "figure 1": {
      "source_image": "relative/or/absolute/path.png",
      "panels": {
        "A": "relative/or/absolute/panel.png"
      }
    }
  }
}
```

Validation rules:

- JSON parses.
- Top-level value is an object.
- At least one nested mapping exists when selected figure/table rows exist.
- Any local path fields found in values should resolve to existing files or be logged as missing.

## `excel_block_inventory.csv`

Minimum expectations:

- CSV parses.
- Each row describes one block or table candidate.
- Required useful columns when generated by current code:
  - `excel_file`
  - `excel_sheet`
  - `block_id`
  - `group_id`
  - `element_id`
  - `block_csv_path`
  - `block_meta_path`
  - `block_type`

Validation rules:

- Non-empty `block_csv_path` resolves under the paper folder.
- Block CSV files parse as CSV.
- `Exp_Excel_Blocks/` exists when rows exist.

## `three_core_result_all.json`

Minimum expectations:

- JSON parses.
- Top-level value is a list.
- Rows include `excel_file`, `excel_sheet`, and `result` when produced by current code.

## `separated_panels_gemini/`

Minimum expectations:

- Directory exists when panel separation is performed.
- Contains image files or subfolders with image files.
- Recorded panel paths in mapping JSON exist.

## `excel_mapping.json`

Expected shape:

```json
{
  "figure 1": [
    {
      "pdf_item_id": "figure 1",
      "excel_item_id": "table or block id",
      "excel_file": "source.xlsx",
      "excel_sheet": "Sheet1",
      "block_id": "block id",
      "block_csv_path": "Exp_Excel_Blocks/source/Sheet1/block.csv",
      "reason": "short rationale"
    }
  ]
}
```

Validation rules:

- JSON parses.
- Top-level value is an object.
- Non-empty `block_csv_path` values resolve under the paper folder.

## `excel_mapping_rows.csv`

Minimum expectations:

- CSV parses.
- Useful columns:
  - `pdf_item_id`
  - `excel_item_id`
  - `excel_file`
  - `excel_sheet`
  - `block_id`
  - `block_csv_path`
  - `reason`
