# Troubleshooting

## Legacy context accidentally treated as runtime code

Cause:
- An external-agent task read `agent_workspace/legacy_context/` and then tried to execute or import a Gemini/API-dependent reference script.
- The context copy is stale after original legacy folders changed.

Fix:
- Treat `agent_workspace/legacy_context/` as read-only reference only.
- Follow `AGENT_INSTRUCTIONS.md`, `PIPELINE_SPEC.md`, `STAGE_CONTRACTS.md`, and `OUTPUT_SCHEMA.md`; they override legacy behavior.
- Do not execute or import Gemini/API-dependent legacy scripts unless the operator explicitly requests legacy mode.
- Regenerate the context copy with `python agent_workspace/tools/build_legacy_context.py` when needed.

## find_api ModuleNotFoundError

Symptom:
- `ModuleNotFoundError: No module named 'find_api'`
- The failure appears while running `03_figure_mapping`, `03_split_excel_blocks_batch`, `04_figure_separate`, or `04_ft_excel_matcher`.

Cause:
- A legacy Gemini/API script was imported.
- This should not happen in API-free `external_agent` or `heuristic` mode.

Fix:
- Check `STAGE_EXECUTION_MODE` in `Agent_Task_Runner.py`.
- Set the stage mode to `external_agent` or `heuristic`.
- Do not run `legacy` mode unless `find_api.py`, old Gemini helper modules, and credentials are available.

## Legacy Script Uses Hard-Coded Path

Symptom:
- Script runs but processes an unrelated folder.
- Output appears under an old `/Users/kogeon/...` or `C:\Users\kogun\...Extraction_Examples...` path.

Action:
- Prefer `Agent_Task_Runner.py`.
- If direct execution is necessary, modify only the execution block or call the script functions via import.
- Log the exact path used.

## API Credentials Missing

Symptom:
- `vertex.json`, `gemini_api.txt`, or service account errors.
- Project ID missing.

Action:
- Do not hard-code secrets.
- Check whether credential files exist where `find_api.py` expects them.
- If credentials are unavailable, mark the task blocked with the missing credential name.

Action:

- Log the batch job ID and retry reason.

## Marker image wrongly cropped

Cause:
- Marker extracted incomplete or mismatched figure image.

Fix:
- Mark entry as `source_quality="suspect_crop"`.
- Render original PDF page using PyMuPDF.
- Save under `pdf_page_renders/`.
- Use `selected_source_for_paneling`.
- If uncertain, set `manual_required=true`.

## unified_extraction.csv empty

Cause:
- No selected figure/table rows were available, or no selected item could be mapped to usable evidence.

Fix:
- Check `fig_table_lnpdb_classified.csv` and `.manual_select_review_done`.
- Validate `excel_mapping.json` and `excel_block_inventory.csv`.
- Run `04_ft_excel_matcher` before `06_unified_lnpdb_extraction`.
- If evidence is genuinely missing, create review flags and keep `manual_required=true`.

## Missing Excel block mapping

Cause:
- A selected figure/table item has no matching `block_csv_path`, or the mapped CSV path is stale.

Fix:
- Revalidate `excel_block_inventory.csv`.
- Confirm files exist under `Exp_Excel_Blocks/`.
- Rerun or repair `04_ft_excel_matcher`.
- Do not extract numeric values from memory or captions when the authoritative Excel block is missing.

## Formulation/value mismatch

Cause:
- Formulation labels, legend groups, or table rows do not line up cleanly with extracted numeric values.

Fix:
- Use figure image/panel evidence for labels and legend interpretation.
- Use markdown captions and methods for formulation descriptions.
- Add rows to `unified_extraction_review_flags.csv`.
- Leave uncertain formulation fields blank and set `manual_required=true`.

## SMILES unresolved

Current unified extraction outputs intentionally force `IL_SMILES`, `HL_SMILES`, `CHL_SMILES`, `PEG_SMILES`, and `Fifth_component_SMILES` blank. Blank SMILES in `unified_extraction.csv`, `unified_extraction_final.csv`, or `unified_extraction_lnpdb_like.csv` are expected and should not be treated as a manual-review issue.

Cause:
- Compound name is ambiguous, unavailable in deterministic lookup tools, or only shown as an image.

Fix:
- Check `compound_inventory_standardized.csv`, `smiles_resolved.csv`, and `smiles_resolution_qc.csv`.
- Use OPSIN, PubChem, CIR, local LNPDB references, text/IUPAC deterministic lookup, or manually curated/manual-verified SMILES files only for Stage 05 QC artifacts.
- Do not use DECIMER, MolScribe, `worker_mol.py`, molecule image crops, image recognition, `recognition.py`, or `segmentation.py` in the active workflow.
- Do not project SMILES into current unified extraction outputs.
- Preserve component names and molar ratios.
- Do not set `manual_required=true` solely because output SMILES columns are blank.

## Figure evidence missing

Cause:
- `source_image`, `selected_source_for_paneling`, or `fallback_render` is missing or invalid.

Fix:
- Validate `total_figure_mapping.json`.
- If Marker image is suspect, render the PDF page fallback.
- Record `evidence_image` only when the path exists and supports the extracted interpretation.

## CLI agent hallucinated fields

Cause:
- A condition or formulation field was inferred without supporting Excel, markdown, image, or PDF evidence.
- Stage 06 populated experimental numeric assay/readout columns without reliable Excel/source-data provenance.
- A value appears to come from graph image digitization, pixel/axis extraction, visual estimation, heatmap color, caption-only inference, or hallucination.

Fix:
- Remove unsupported condition/formulation fields.
- Keep `metric_type`, `original_values`, `aggregated_value`, `unit`, and `replicate_type` only when backed by mapped Excel/source-data blocks.
- Add `evidence_excel`, `excel_file`, `excel_sheet`, or `block_csv_path` for populated value rows.
- Leave value columns blank when no reliable Excel/source-data mapping exists.
- Add `evidence_text`, `evidence_excel`, or `evidence_image` for condition/formulation fields.
- Set `confidence="low"` and `manual_required=true` when evidence is incomplete.

## Legacy Gemini script accidentally imported

Cause:
- A stage was run in `legacy` mode or a task used scripts from `1_Extract_Exp_Figs`, `2_Extract_SMILES`, `3_Extract_Formula_by_Figs`, or `4_Extract_Exp_Vals` directly.

Fix:
- Use `external_agent` or `heuristic` mode for active stages.
- Do not import `find_api.py`, `LLM_API.py`, or `LLM_Batch.py`.
- Keep legacy scripts for audit/manual legacy mode only.

## External CLI command not found

Cause:
- `run-agent-active` tried to launch `codex`, `claude`, or a custom command that is not on `PATH`.

Fix:
- Confirm the CLI is installed and available from the same shell.
- Use `--agent custom --agent-command "<COMMAND_TEMPLATE>"` with the full executable path if needed.
- Check the log event `external_agent_call` for `stderr_tail`.

## Agent command returned non-zero

Cause:
- The external CLI agent process exited with a non-zero return code.

Fix:
- Inspect `stdout_tail`, `stderr_tail`, and `returncode` in `agent_workspace/logs/<paper>.jsonl`.
- Re-run with `--max-agent-retries` after fixing command configuration.
- Use `--continue-on-error` only when downstream stages can still provide useful diagnostics.

## Validation retry failed

Cause:
- The external CLI agent ran, but generated outputs still did not pass `validate_stage`.

Fix:
- Open the stage task file under `agent_workspace/tasks/`.
- Read the appended `Validation Failure Feedback` section.
- Fix the listed missing files, columns, paths, or row counts.
- Do not ignore validation failure or proceed with unvalidated final outputs.

## Codex/Claude keeps asking for approval

Cause:
- The selected CLI agent is running in an interactive approval mode that blocks subprocess automation.

Fix:
- Configure the CLI for noninteractive execution before using `run-agent-active`.
- For custom Codex runs, consider a command template such as `codex exec --full-auto "{prompt_text}"` when appropriate for the environment.
- Keep file edits scoped to the paper folder and repository workspace.

## Custom command quoting issue on Windows

Cause:
- The custom command template mixes nested quotes, spaces, Korean paths, or placeholders in a way the subprocess tokenizer cannot split as intended.

Fix:
- Prefer stdin mode by ending the command with `-` or using `{prompt_stdin}`.
- Put `{prompt_file}`, `{stage}`, and `{paper_folder}` as separate quoted arguments when needed.
- Prefer `--agent-command "cmd /c codex exec --cd \"{project_root}\" --dangerously-bypass-approvals-and-sandbox --add-dir \"{paper_folder}\" -"`.
- Check the logged `command` field; it redacts prompt text as `<prompt_text>` but preserves argument structure.
